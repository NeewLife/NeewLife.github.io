Hexo 설치

nodejs 홈페이지에서

왼쪽꺼 다운로드

만약 사이트가 바뀌어서 헷갈리면 LTS 버전으로 다운로드

LTS 는 안정적으로 테스트까지 완료한 배포버전이고
최신기능은 아직 실험중인 베타테스팅 개념이라고 생각하면 될듯

설치시 Add to PATH 선택 후 다운

  • 명령 프롬포트(cmd) 진입
  • 설치 완료 후 cmd 에서 node -v 입력

npm install -g hexo-cli 입력 후 엔터

블로그 만들기

  • 바탕화면에서 git bash 후 hexo init blog 입력

  • cd blog —> ls 차례대로 입력

  • github 에서 0616에서 배웠던 것들 업로드 한다. (git add . 로 모든 파일 업로드)
  • 바탕화면에 blog 폴더 우클릭 후 PyCharm 으로 열기

사실 PyCharm 을 쓰던 IntelliJ 를 쓰던 상관없다.
요즘은 PyCharm이 너무 무거운거 같아서 Intellij로 시도중임

  • 터미널 열고

npm install

npm install hexo-server –save

npm install hexo-deployer-git –save 차례대로 입력

hexo server 입력

  • 하이퍼링크 클릭하면 블로그 완성

블로그 배포하기

  • github 들어가서 새 Repository 로 “본인닉네임.github.io” 로 만들기

  • blog → _config.yml 파일로 들어가서(themes 안에 있는 yml 파일 X)

저렇게 쓰기(NeewLife 자리에 본인 닉네임)

  • 맨 밑에 내려와서 deploy에 다음과 같이 적어준다

https://github.com/본인닉네임/본인닉네임.github.io.git 임

branch 는 자신의 branch 이름에 맞게

  • 다시 깃배쉬 밑에 “hexo generate, hexo deploy” 입력(단축키로 hexo d -g 로 실행해도 된다.)

hexo generate 는 public 이라는 폴더에 프로토타입을 만들어 자기 로컬 폴더에 변경사항 적용시키는 것이고
hexo deploy 는 아까 만든 github.io 서버에 배포해서 모든 사람들에게 변경사항이 적용되게 한다.

hexo server(단축키 hexo s) 는 프로토타입도 아닌 와이어프레임? 같은 개념으로 이해하면 될듯
주로 hexo server 를 써서 빠르게 테스트 해볼 수 있다.

  • 주소창에 “본인닉네임.github.io” 검색하면 Hexo 블로그 배포완료된걸 확인

  • 다 됐으면 git add . 로 업로드

주의사항

_config.yml 파일은 함부로 건들지 말자

홈페이지 제목을 변경하고 싶으면 title을,

내 정보를 바꾸고 싶다면 author 를,

언어를 바꾸고 싶다면 language 를 ko 로 변경하면 된다.

그 외 테마변경, 한 페이지당 게시물 출력수 등 변경 할 수 있긴하다.
