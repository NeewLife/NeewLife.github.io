조건문 - if, else, switch-case, 삼항연산자

if문만 잘 써도 NullPointException 에러 90%는 잡을 수 있다!

String을 비교할 때 정적 데이터와 동적 데이터를 비교할때는 정적데이터.equals(동적데이터); (정적데이터가 앞으로 나와야됨)
왜 why?
웹에서는 외부에서 데이터를 받아올 수 있다. 근데 만약 동적데이터.equals(정적데이터); 이렇게 하면 정적데이터는 하드코딩된 데이터인데
동적데이터는 사용자에게 받아오는 값이므로 무슨 값이 들어올 지 모르므로 NullPointException이 발생할 수 있다.

if문을 쓸땐 맨 위의 조건에 null 체크를 해야된다. (값 밸리데이션 인듯. if, else-if로 안되는 값들 거르고 내려가도록 하는 것)
if(“”.equals(str) || str == null){
}else-if(“melon”.equals(fruit)){
}else-if(“apple”.equals(fruit){
} ….

if else를 짤땐 주체가 같고 값이 다를 때 쓴다.
if(“사과”.equals(str)){
} 하고 갑자기 뜬금없이 else-if(“사과” >= 10000){
} 이렇게 하면 안된다.
