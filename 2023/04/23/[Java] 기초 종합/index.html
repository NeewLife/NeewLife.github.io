[Java] 기초 종합

2023/04/23 카테고리 Java

변수(Variable)

데이터를 저장하기 위해 프로그램으로 할당받은 메모리 공간

  • 변수 선언 = 프로그래밍을 하기 위해 변수를 선언 해주는 것(반드시 데이터타입을 명시 해주어야 한다.)
  • 변수 초기화 = 선언된 변수에 최초로 값을 넣어서 저장해주는 것(리셋이 아니다)

객체(object)

효율적으로 정보를 관리하기 위해 사람들이 의미를 부여하고 분류하는 논리적인 단위

프로그래밍에서는 클래스에 정의된 내용대로 메모리에 생성된 것을 말함.

클래스(class)

클래스란 어떤 특정 종류의 모든 객체들에 대해 일반적으로 적용할 수 있는 멤버 변수와 메소드를 정의한 소프트웨어적 설계도 또는 프로토타입(Prototype)이라 할 수 있다.

  • 프로토타입 : 정보시스템의 미완성 버전 혹은 중요한 기능들이 포함되어 있는 시스템의 초기모델이다.

즉, 실세계에 존재하는 객체들이 가질 수 있는 상태와 행동들에 대해 소프트웨어적으로 추상화(abstraction) 해 놓은 것이다.

인스턴스

클래스로부터 객체를 만들어 내는 과정을 클래스의 인스턴스화 라고 하며,
어떤 클래스로부터 만들어진 객체를 클래스의 인스턴스 라고 한다.

인스턴스는 객체를 소프트웨어에 실체화한것 이라고 말할 수 있다.

실체화된 객체(인스턴스)는 메모리에 할당된다.

클래스는 설계도/객체는 설계도로 구현한 모든 대상

생성자

인스턴스가 호출될때 호출되는 ‘인스턴스 초기화 메서드’ 이다.

인스턴스 변수의 초기화에 사용되고, 몇가지 특징을 제외하면 메서드와 다를게 거의 없다.

  • 생성자는 리턴 값이 없다. void 또한 사용하지 않는다.
  • 생성자의 이름은 반드시 클래스명과 동일해야 한다.
  • 생성자는 메서드처럼 클래스 내에 선언되고, 오버로딩이 가능하므로 하나의 클래스 안에 여러개의 생성자가 존재할 수 있다.
  • 생성자를 설정 안해도 하나의 클래스에는 기본적으로 기본 생성자가 들어가있다.

객체지향 프로그래밍(Object-Oriented Programming, OOP)

프로그래밍에서 필요한 데이터들을 추상화 시켜 상태와 행위를 가진 객체 로 만들고, 객체들간의 상호작용을 통해 로직을 구성하는 프로그래밍 방법이다.

예를 들어 객체가 레고 조각들이라면 그 레고들을 조립해서 만드는 것이 객체지향 프로그래밍이라고 할 수 있다.

객체지향 프로그래밍의 5개 키워드

  1. 클래스 + 인스턴스
  2. 추상화
    1. 현실적으로 불필요한 부분들을 생략하고 객체의 속성중 중요한 것에만 중점을 두어 개략화 하는 것
      예) 지하철 노선도를 추상화 한다면
      위성사진 → 지적도 → 실제 노선도 → 우리가 보는 지하철 노선도
  3. 캡슐화
    1. 기능과 특성의 모음을 “클래스” 라는 캡슐에 분류해서 넣는 것이 캡슐화 이다.
    2. 객체가 외부에 노출되지 않아야 할 정보 또는 기능을 접근제어자를 통해서 적절히 제어 권한이 있는 객체에서만 접근하도록 하기 위해서 쓴다.
  4. 상속
    1. 부모클래스의 속성과 기능을 그대로 이어받아 사용할 수 있게 하고, 기능의 일부분을 변경해야 할 경우 상속받은 자식클래스에서 해당 기능만 다시 수정(정의)하여 사용할 수 있게 하는 것이다.
    2. 다중상속은 불가능하다. (클래스간의 혼란을 줄 수 있기 때문에 반드시 하나의 상속만 가능하고, 필요에 따라서 인터페이스를 사용한다.)
  5. 다형성
    1. 하나의 변수명, 함수명이 상황에 따라 다른 의미로 해석될 수 있는 것을 의미한다.
      • 오버로딩 : 부모클래스의 메서드와 같은 이름, 매개변수를 새로 재정의 하는 것
      • 오버라이딩 : 같은 이름의 함수를 여러개 정의하고, 매개변수의 타입과 개수를 다르게 하여 매개변수에 따라 다르게 호출할 수 있게 하는 것

  • 객체지향 프로그래밍의 장단점

  • 장점

    • 코드 재사용이 용이하다
      다른 사람이이 만든 클래스를 가져와서 이용할 수 있고 상속을 통해 확장해서 사용할 수 있다.
    • 유지보수가 쉽다.
      수정해야할 부분들이 클래스 내부에 멤버 변수 혹은 메서드로 존재하기 때문에 해당 부분만 수정하면 된다.
    • 대형 프로젝트에 적합
      클래스 단위로 모듈화 시켜서 개발할 수 있으므로 대형 프로젝트처럼 여러 명, 여러 회사에서 프로젝트를 개발할 때 업무 분담하기 쉽다.

  • 단점

    • 처리속도가 느리다.
    • 객체가 많으면 용량이 커질 수 있다.
    • 설계시에 많은 시간과 노력이 필요하다.

추상클래스와 인터페이스

추상클래스

  • 실체클래스의 공통적인 부분(변수, 메서드)을 추출해서 선언한 클래스 즉, 실체클래스를 추상화 한 클래스
  • 따라서 실체클래스와는 상속관계에 있다.
  • 추상클래스는 객체를 만들 만큼의 실체성과 구체성이 없기 때문에 객체 생성이 불가하다.

예) A키보드는 적축, B키보드는 갈축, C키보드는 청축이라고 할 때 소리는 다 다르지만, 기계식 키보드라는 공통점이 있다.

그러므로 ‘기계식 키보드’ 라는 추상클래스 안에 A키보드 B키보드 C키보드 등등 이 상속되어 있는 개념이다.

A 키보드라는 객체를 만든다고 하면 ‘기계식 키보드’ 추상클래스를 상속 받아서 객체를 생성한다. 그 안에서 Override를 하든 새로운 A키보드만의 기능을 만들든 상관없지만,

A키보드 B키보드 C키보드 모두 기계식 키보드라는 공통 특성을 가지게 된 객체이다.

왜 사용하는가?

  1. 공통된 필드와 메서드를 통일할 목적으로
  2. 실체클래스를 구현하게 될 시 시간절약 하기 위해
  3. 추상클래스라는 공통규격 안의 실체클래스를 구현하기 위해 사용한다.

인터페이스

  • 극단적으로 동일한 목적 하에 동일한 기능을 강제하는 것
  • 자바의 다형성을 이용하여 개발코드 수정을 줄이고 유지보수성을 높인다.
  • 인터페이스 끼리는 다중상속이 가능하다.

왜 사용하는가?

  1. 상속은 하나의 상속밖에 안되지만 인터페이스를 사용하면 다중상속이 가능하다.
  2. 대규모 프로젝트 개발 시 일관되고 정형화된 개발이 가능하다.

추상클래스와 인터페이스 공통점과 차이점

공통점

  • 메서드의 선언만 있고 구현내용은 없다. (추상메서드)
  • 객체를 생성할 수 없으며, 상속받은 실체클래스가 반드시 선언된 추상메서드를 사용해야한다.

차이점

  • 추상클래스는 다중상속이 불가능 / 인터페이스는 다중상속 가능
  • 추상클래스는 클래스처럼 일반 변수, 생성자, 일반메서드, 추상메서드를 가질 수 있지만 /
    인터페이스는 상수와 추상메서드만 가질 수 있다.
    (자바 8부터 인터페이스에서 default method로 구현이 가능하지만 일반적으로 구현은 없다.)
  • 추상클래스는 그대로 자신의 기능들을 하위로 확장(extends)시키는 느낌(부모의 유전자를 물려받는다.)
    인터페이스는 인터페이스에 정의된 메서드를 각 클래스의 목적에 맞게 동일한 기능으로
    (implements)구현하는 느낌.
    (부모로부터 유전자를 물려받는 것이 아니라 사교적으로 필요에 따라 결합하는 관계)즉, 구현 객체가 같은 동작을 한다는 것을 보장하기 위함.
